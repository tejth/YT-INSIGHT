## вңҰ What is YT Insight?

**YT Insight** is a Chrome Extension + Flask backend that lets you have a full AI conversation about any YouTube video. Paste a video URL, hit Load, and ask anything вҖ” summaries, key points, specific concepts explained, timestamps вҖ” all answered from the actual video transcript using a RAG (Retrieval-Augmented Generation) pipeline.

> Built as a college project using **LangChain + HuggingFace** instead of OpenAI вҖ” completely free to run.

## вңҰ Tech Stack

| Layer | Technology | Purpose |
|-------|-----------|---------|
| **Chrome Extension** | Manifest V3, Vanilla JS | UI popup + service worker |
| **Backend** | Python 3.10+, Flask, Flask-CORS | REST API server |
| **Transcript** | `youtube-transcript-api` v0.7+ | Fetch video captions |
| **Chunking** | `langchain-text-splitters` | Split transcript into 800-char chunks |
| **Embeddings** | HF Inference API В· `all-MiniLM-L6-v2` | Semantic vector search |
| **Vector Search** | NumPy cosine similarity | Find relevant chunks |
| **LLM** | HF Router В· `Llama-3.1-8B:cerebras` | Generate answers |
| **Zero Downloads** | Pure API calls | No local model files |

## вңҰ Setup Guide

### Prerequisites

- Python 3.10 or higher
- Google Chrome browser
- A HuggingFace account (free)

---

### Step 1 вҖ” Get HuggingFace API Token

1. Go to вҶ’ [huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)
2. Click **New Token** вҶ’ name it anything вҶ’ select **Read** permission
3. Copy the token (starts with `hf_...`)

---

### Step 2 вҖ” Install Python Dependencies

```bash
pip install flask flask-cors youtube-transcript-api langchain-text-splitters numpy requests
```

---

### Step 3 вҖ” Start the Backend Server

**Windows (PowerShell):**
```powershell
$env:HF_API_KEY = "hf_your_token_here"
python server.py
```

**Mac / Linux:**
```bash
export HF_API_KEY="hf_your_token_here"
python server.py
```

You should see:
```
рҹҡҖ YT Insight backend running on http://localhost:5000
```

> Keep this terminal open while using the extension.

---

### Step 4 вҖ” Load the Chrome Extension

1. Open Chrome вҶ’ go to `chrome://extensions/`
2. Enable **Developer Mode** (toggle, top-right)
3. Click **Load unpacked**
4. Select the `yt-rag-extension/` folder
5. Pin **YT Insight** from the extensions toolbar

---

### Step 5 вҖ” Use It!

1. Open any YouTube video
2. Click the **YT Insight** icon вҖ” URL auto-fills
3. Click **LOAD VIDEO** and wait ~5вҖ“15 seconds
4. Ask anything about the video

## вңҰ API Reference

The Flask backend exposes 3 endpoints:

### `GET /health`
Check if the server is running.
```json
{ "status": "ok" }
```

---

### `POST /load`
Load a YouTube video and build the RAG index.

**Request:**
```json
{ "url": "https://www.youtube.com/watch?v=Gfr50f6ZBvo" }
```

**Response:**
```json
{
  "video_id": "Gfr50f6ZBvo",
  "chunk_count": 87,
  "preview": "In this video we explore the history of DeepMind...",
  "cached": false
}
```

---

### `POST /ask`
Ask a question about a loaded video.

**Request:**
```json
{
  "video_id": "Gfr50f6ZBvo",
  "question": "What is AlphaFold?"
}
```

**Response:**
```json
{
  "answer": "AlphaFold is an AI system developed by DeepMind that predicts...",
  "sources": [
    "...AlphaFold represents a major breakthrough in protein structure...",
    "...the system achieved human-expert level accuracy on the CASP14..."
  ]
}
```

---

### `GET /transcript?video_id=...`
Get the full raw transcript text.

## вңҰ How RAG Works (Simple Explanation)

```
Your Question
     в”Ӯ
     в–ј
  Embed question вҶ’ [0.23, -0.87, 0.41, ...]  (384 numbers)
     в”Ӯ
     в–ј
  Compare with all chunk embeddings using cosine similarity
     в”Ӯ
     в–ј
  Pick top 4 most relevant chunks from transcript
     в”Ӯ
     в–ј
  Send chunks + question to LLM:
  "Answer this question using ONLY this context: ..."
     в”Ӯ
     в–ј
  Answer вң“
```

This means the AI **cannot hallucinate** facts not in the video вҖ” it only uses what's actually said.

## вңҰ Troubleshooting

| Error | Cause | Fix |
|-------|-------|-----|
| рҹ”ҙ Server offline (red dot) | Flask not running | Run `python server.py` |
| вҡ  No transcript available | Video has no captions | Try a video with CC enabled |
| вҡ  404 Embedding error | Wrong HF endpoint | Check `EMBED_MODEL` in server.py |
| вҡ  400 LLM error | Model format wrong | Ensure model name has `:cerebras` suffix |
| Model stuck at 0% | Windows symlink issue | Use `$env:HF_HOME = "D:\hf_cache"` |
| `export` not recognized | You're on PowerShell | Use `$env:HF_API_KEY = "..."` |
| `langchain.text_splitter` not found | Old LangChain | Use `langchain_text_splitters` |

## вңҰ Customization

### Change the LLM Model
In `server.py`, edit:
```python
LLM_MODEL = "meta-llama/Llama-3.1-8B-Instruct"
```

Other fast free options:
```python
LLM_MODEL = "Qwen/Qwen2.5-7B-Instruct"       # Great for multilingual
LLM_MODEL = "google/gemma-2-2b-it"            # Very lightweight
LLM_MODEL = "mistralai/Mistral-7B-Instruct-v0.3"  # Good reasoning
```

When calling, append `:cerebras` or `:auto`:
```python
"model": f"{LLM_MODEL}:cerebras"   # Fast Cerebras hardware
"model": f"{LLM_MODEL}:auto"       # HF auto-picks best provider
```

### Change Chunk Size
```python
# Smaller = more precise, more API calls
# Larger = more context per chunk, fewer calls
RecursiveCharacterTextSplitter(chunk_size=800, chunk_overlap=150)
```

## вңҰ Limitations

- Videos must have **English captions** (auto-generated captions work too)
- Free HuggingFace tier has **rate limits** вҖ” long videos may be slow
- Video index is stored **in memory** вҖ” restarting server clears the cache
- Works only while the **Flask server is running** locally
